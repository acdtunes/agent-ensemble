# Job Stories: Conflict Detection

## Primary Job Story

**When** I'm managing a team of AI agents working on a feature in parallel,
**I want to** see which kanban cards will conflict due to shared files,
**so I can** plan worktree usage and avoid unexpected merge conflicts during integration.

### Job Dimensions

| Dimension | Description |
|-----------|-------------|
| **Functional** | Identify file overlaps between cards; know which cards need worktrees |
| **Emotional** | Feel confident that parallel work won't lead to integration chaos |
| **Social** | Demonstrate effective team coordination; avoid blame for merge disasters |

## Secondary Job Story

**When** I'm reviewing the kanban board before assigning work,
**I want to** see which pending cards would conflict with in-progress work,
**so I can** decide whether to wait, use a worktree, or reassign work strategically.

### Job Dimensions

| Dimension | Description |
|-----------|-------------|
| **Functional** | Preview conflict impact before starting work |
| **Emotional** | Feel in control of work sequencing |
| **Social** | Make informed decisions that teammates can trust |

## Tertiary Job Story

**When** reviewing completed work,
**I want to** understand which cards overlapped (historical conflicts),
**so I can** identify patterns and improve future planning.

> **Note**: This job is deprioritized — conflicts are hidden when all cards are done.
